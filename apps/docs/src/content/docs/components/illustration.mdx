### Usage

```tsx
import { Illustration } from '~/components/ui/illustration';

function Example() {
  return (
    <View className='flex-1 justify-center items-center p-6 gap-12'>
      {/* Basic usage */}
      <Illustration name="Heart" size="medium" />
      
      {/* Different sizes */}
      <View className="flex-row gap-4">
        <Illustration name="Apple_Vision" size="small" />
        <Illustration name="Apple_Vision" size="medium" />
        <Illustration name="Apple_Vision" size="large" />
      </View>
      
      {/* With accessibility */}
      <Illustration 
        name="Coffee" 
        size="large"
        accessibilityLabel="Coffee cup illustration"
        accessibilityHint="Decorative coffee illustration"
      />
      
      {/* Decorative (hidden from screen readers) */}
      <Illustration 
        name="Heart" 
        size="medium"
        decorative={true}
      />
    </View>
  );
}
```

## Size Guide

- **Small**: 80x80 pixels - Perfect for inline usage, list items
- **Medium**: 120x120 pixels - Great for cards, feature highlights  
- **Large**: 160x160 pixels - Ideal for hero sections, empty states

## Accessibility

The Illustration component is built with accessibility in mind:

- **Screen Reader Support**: Automatic or custom `accessibilityLabel` generation
- **Decorative Mode**: Use `decorative={true}` to hide purely decorative illustrations
- **Semantic Roles**: Proper image role assignment for meaningful content
- **Hint Support**: Additional context via `accessibilityHint`

## Best Practices

1. **Use descriptive names**: `accessibilityLabel="Coffee cup illustration"` instead of just "Coffee"
2. **Mark decorative content**: Set `decorative={true}` for purely visual elements
3. **Choose appropriate sizes**: Small for lists, Medium for cards, Large for hero sections
4. **Provide context**: Use `accessibilityHint` for additional context when needed

## Technical Notes

- All illustrations are statically imported for optimal Expo/React Native bundling
- Images are optimized PNG format for crisp rendering at all sizes
- Component uses `resizeMode: 'contain'` to maintain aspect ratios
- TypeScript support with full type safety for illustration names
